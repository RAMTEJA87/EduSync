PROJECT: EduSync AI — Intelligent Adaptive Learning Platform
=============================================================

WHAT IT IS:
A full-stack web app for colleges. It uses AI (Groq LLM) to generate quizzes, detect student weaknesses, predict academic risk, and provide study recommendations. Three roles: Student, Teacher, Admin.

TECH STACK:
- Frontend: React 18 + Vite + TailwindCSS + Framer Motion + Recharts + Lucide Icons
- Backend: Node.js + Express.js + Socket.IO
- Database: MongoDB (Mongoose ODM)
- Auth: JWT tokens + bcryptjs password hashing
- AI: Groq API using llama-3.3-70b-versatile model
- File uploads: Multer (stored in MongoDB as binary data)

PROJECT STRUCTURE:

HOW EACH ROLE WORKS:
---------------------

ADMIN:
- Two-tab dashboard: "Academic Hierarchy" and "User Directory"
- Academic Hierarchy: CRUD for Year/Branch/Section nodes (e.g. Year 2 / CSE / Section A)
- User Directory: Create/edit/delete Students and Teachers; assign students to a section
- API calls: /api/admin/academic, /api/admin/users, /api/admin/users/bulk-import

TEACHER:
- Dropdown to select a class section (Year/Branch/Section)
- KPI cards: Total Students, Active Quizzes, High-Risk Count, Avg Accuracy
- Radar chart showing class topic mastery
- High-risk student panel (auto-flagged by AI)
- "Generate AI Quiz" modal: teacher types a topic + difficulty + num questions OR uploads a PDF; backend sends to Groq to generate MCQs, saves to DB, publishes to the section
- "Upload Notes" modal: teacher uploads PDF/image for a section; students in that section can view it
- Lists of recent quizzes and class notes with delete options
- Comprehensive student analytics table (roll no., accuracy, weakness, risk)
- API calls: /api/academic/* and /api/materials/* (routes partially MISSING from backend)

STUDENT:
- Dashboard fetches from /api/student/dashboard (NOT YET BUILT)
- Shows: welcome header, AI tool cards, area chart of quiz accuracy over time, available quizzes list, quiz history, risk meter, class materials
- Risk Meter: circular gauge shows the student's risk score (%) and label LOW/MEDIUM/HIGH
- AI tool cards link to: /student/youtube (YouTube AI), /student/doubt-solver, /student/smart-revision (NONE of these pages exist yet)
- Clicking a quiz navigates to /student/quiz/:id

QUIZ ATTEMPT FLOW:
- GET /api/quiz/:id/attempt → returns shuffled questions, options shuffled, correct answer NOT sent to client
- Timer = 2 mins per question, auto-submits on timeout
- Student picks options, clicks Next; on last question clicks Submit
- POST /api/quiz/:id/submit sends { answers: [{questionId, selectedOptionText, timeSpent}], timeTakenSeconds }
- Backend scores answers, calls weakAreaDetector (updates weakTopics in User), calls predictionEngine (recalculates risk), calls evaluateAssignment (returns marks)
- Result screen shows: accuracy %, marks, list of weak topics, YouTube search links for each weakness

RISK PREDICTION ALGORITHM (predictionEngine.js):
  baseRisk = 0
  Fetch last 3 quiz results:
    if avg accuracy < 50% → baseRisk += 40
    if avg accuracy < 70% → baseRisk += 20
  Engagement (mocked at 60%): if < 30% → baseRisk += 30
  If student has >1 topic with failureCount >= 2 → baseRisk += 30
  baseRisk >= 75 → HIGH
  baseRisk >= 40 → MEDIUM
  else → LOW
