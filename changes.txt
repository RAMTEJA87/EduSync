WHAT IS MISSING - GAPS MAPPED TO PROBLEM STATEMENT
====================================================================================

---------------------------------------------------------------------
GAP 1: LOGIN IS NOT CONNECTED TO BACKEND
---------------------------------------------------------------------
Problem: Login form bypasses the API. Token is never stored.
Impact: Every feature that needs auth (student dashboard, quiz attempt, etc.) 
        is broken in a real multi-user scenario.
Fix needed:
  - /client/src/pages/auth/Login.jsx: Call POST /api/auth/login on submit
  - Store token + user object in localStorage
  - Redirect to correct dashboard based on role in the response

---------------------------------------------------------------------
GAP 2: STUDENT DASHBOARD HAS NO BACKEND API
---------------------------------------------------------------------
Problem: GET /api/student/dashboard does not exist.
Impact: Student dashboard always shows "Failed to load dashboard."
Fix needed:
  - Create route: GET /api/student/dashboard (protected, STUDENT only)
  - Aggregate and return:
      * user profile (name, role, contextString, topWeakness)
      * progressionData (last N quiz results mapped to chart format)
      * availableQuizzes (quizzes assigned to student's section, not yet attempted)
      * riskMetrics (score + label from User.overallRiskLevel)
      * recommendedResources (class materials for student's section)
      * quizHistory (past quiz results)

---------------------------------------------------------------------
GAP 3: TEACHER ANALYTICS API MISSING
---------------------------------------------------------------------
Problem: GET /api/academic/:id/analytics and GET /api/academic/public do not exist.
Impact: Teacher dashboard shows no data, stays at zeros.
Fix needed:
  - GET /api/academic/public → returns all AcademicStructure documents
  - GET /api/academic/:id/analytics → returns:
      * totalStudents, activeQuizzes, highRiskCount, avgAccuracy
      * radarData (topic-wise failure counts for radar chart)
      * highRiskStudents (name, failedTopic, riskLevel)
      * recentQuizzes (last N quizzes for this section)
      * recentMaterials (uploaded notes for this section)
      * allStudents (table: rollNumber, name, avgAccuracy, weakness, riskLevel)

---------------------------------------------------------------------
GAP 4: ADMIN CRUD ROUTES DO NOT EXIST
---------------------------------------------------------------------
Problem: /api/admin/academic and /api/admin/users have no backend routes.
Impact: Admin panel cannot create sections or users.
Fix needed:
  - GET    /api/admin/academic → list all structures (with students populated)
  - POST   /api/admin/academic → create a new Year/Branch/Section
  - PUT    /api/admin/academic/:id → update a structure
  - DELETE /api/admin/academic/:id → delete and unassign students
  - GET    /api/admin/users?role=STUDENT|TEACHER → list users by role
  - POST   /api/admin/users → create user (with hashed password)
  - PUT    /api/admin/users/:id → update (name, email, role, academicContext)
  - DELETE /api/admin/users/:id → delete user and their quiz results

---------------------------------------------------------------------
GAP 5: CLASS NOTES (MATERIALS) ROUTES MISSING
---------------------------------------------------------------------
Problem: /api/materials/upload and /api/materials/:id not implemented.
Impact: Teachers cannot upload notes; students cannot access class materials.
Fix needed:
  - Create Material model: { title, fileUrl, mimetype, academicContextId, uploadedBy }
  - POST /api/materials/upload (Teacher) → handle file with Multer, save record
  - DELETE /api/materials/:id (Teacher) → remove file and DB record
  - Include materials in student dashboard API response (filtered by student's section)

---------------------------------------------------------------------
GAP 6: QUIZ DIFFICULTY IS NOT TRULY ADAPTIVE
---------------------------------------------------------------------
Problem stated: Difficulty should adjust dynamically per student performance.
Current state: Teacher sets difficulty manually (EASY/MEDIUM/HARD) at quiz creation.
Fix needed:
  - Add adaptive logic in generateQuiz: if student risk is HIGH, set difficulty to EASY
  - Or generate per-student versions of a quiz based on their weakTopics
  - Use predictionEngine result to recommend appropriate difficulty to teacher

---------------------------------------------------------------------
GAP 7: YOUTUBE AI SUMMARIZER — NOT BUILT
---------------------------------------------------------------------
Problem stated: AI summarizes educational videos into structured notes.
Current state: A card exists on the student dashboard linking to /student/youtube
               but that page does not exist.
Fix needed (full feature):
  - Build /client/src/pages/student/YouTubeAI.jsx page
  - Input: YouTube video URL
  - Backend: GET /api/ai/youtube-summary
    * Use youtube-transcript (npm) to extract transcript
    * Send transcript to Groq to generate: summary, key points, multi-lingual notes
  - Display structured output with timestamp-linked sections
  - Add the route in App.jsx

---------------------------------------------------------------------
GAP 8: AI DOUBT SOLVER (CHATBOT) — NOT BUILT
---------------------------------------------------------------------
Problem stated: 24/7 AI chatbot for student queries, practice questions, revision hints.
Current state: Card exists, /student/doubt-solver page does not exist.
Fix needed:
  - Build /client/src/pages/student/DoubtSolver.jsx (chat UI)
  - Backend: POST /api/ai/doubt → accepts { message, context (student's weak topics) }
  - Use Groq to generate answers contextually aware of student's academic level
  - Maintain conversation history in component state (not persisted for now)
  - Add route in App.jsx

---------------------------------------------------------------------
GAP 9: SMART REVISION GENERATOR — NOT BUILT
---------------------------------------------------------------------
Problem stated: AI generates personalized revision plan targeting weakness areas.
Current state: Card exists, /student/smart-revision page does not exist.
Fix needed:
  - Build /client/src/pages/student/SmartRevision.jsx
  - Backend: GET /api/ai/smart-revision (protected, STUDENT only)
    * Fetch student's weakTopics from DB

  - Add route in App.jsx

---------------------------------------------------------------------
GAP 10: REAL-TIME CLASSROOM FEATURES — NOT BUILT
---------------------------------------------------------------------
Problem stated: Live engagement tracking, sentiment analysis, confusion detection during class.
Current state: Socket.IO is set up on the server but no events are used in the frontend.
Fix needed:
  - Teacher broadcasts a "live session" event for a class section
  - Students join a socket room for their section
  - Real-time quiz polling: teacher sends a quick question; students respond live
  - Aggregate responses shown to teacher in real time (confusion meter)
  - Sentiment analysis: analyze student's typed questions (via Groq) to detect confusion

---------------------------------------------------------------------
GAP 11: STUDENT PERFORMANCE PREDICTION — PARTIALLY BUILT
---------------------------------------------------------------------
Current state: predictionEngine.js exists and correctly calculates risk.
               evaluateRisk() is called after quiz submission.
               But engagement is currently hardcoded at 60% (mocked).
Fix needed:
  - Track actual student engagement: login frequency, time-on-platform, materials viewed
  - Add a lastLoginAt field and loginCount to User model
  - Track material views (increment view count when student opens a PDF)
  - Pass real engagement data into predictionEngine instead of mock value
  - Auto-alert teacher (via Socket.IO or email) when a student flips to HIGH risk

---------------------------------------------------------------------
GAP 12: MULTILINGUAL NOTES — NOT BUILT
---------------------------------------------------------------------
Problem stated: AI converts content to multi-lingual notes.
Fix needed:
  - In YouTube AI summarizer or Smart Revision, add a language selector
  - Pass selected language in Groq prompt to generate notes in that language
  - Frontend: dropdown with common languages (Hindi, Telugu, Tamil, Spanish, etc.)

---------------------------------------------------------------------
GAP 13: AUTH FLOW — ROLE-BASED REDIRECT AFTER LOGIN
---------------------------------------------------------------------
Problem: Login doesn't call the API; there's no protected routing.
Fix needed:
  - Login form calls POST /api/auth/login, stores { token, role } in localStorage
  - Create a ProtectedRoute wrapper component that checks for valid token + role
  - Wrap all dashboard routes with ProtectedRoute (redirect to /login if unauthorized)
  - Auto-redirect on token expiry
